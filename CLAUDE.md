# Finance Manager — CLAUDE.md

## Цель
Личный финансовый менеджер. PWA. React. Мобильный + десктоп.
Одна задача: понимать сколько денег есть сейчас и сколько можно тратить каждый день.

---

## Стек
- React (Vite)
- localStorage для хранения данных
- CSS Modules или Tailwind (на усмотрение, главное — чисто)
- Никакого бэкенда, никаких внешних API

---

## Структура данных

```js
// Текущий баланс (всегда редактируемый вручную)
currentBalance: number

// События календаря
events: [
  {
    id: string,
    date: "YYYY-MM-DD",
    type: "income" | "expense",
    amount: number,
    label: string
  }
]
```

---

## Логика расчётов

### Баланс на конкретный день
```
баланс(day) = currentBalance
  + сумма всех income до и включая day
  - сумма всех expense до и включая day
```

### Сколько можно тратить сегодня
```
доступно(today) = баланс(today) + будущие income до конца месяца
                  - будущие expense до конца месяца
                  / оставшиеся дни в месяце включая сегодня
```

## Экраны и компоненты

### 1. Header (всегда виден)
- Текущий баланс — большая цифра, кликабельная для редактирования
- При клике — инлайн-редактирование, подтверждение по Enter или blur

### 2. Календарь (основной экран)
- Месячный вид, все дни месяца
- Переключение месяцев: стрелки + свайп на мобильном
- На каждом дне:
  - Точки или иконки если есть события
  - Сумма остатка на этот день
- Сегодня — визуально выделен
- Дни с отрицательным балансом — выделены красным

### 3. День (открывается по клику на день)
- Список событий дня (income/expense)
- Кнопка добавить событие
- Итог: остаток на конец дня
- Сколько можно потратить сегодня (только для текущего дня)

### 4. Форма добавления события
- Тип: доход / расход (toggle)
- Сумма (числовой input)
- Подпись (текстовый input, опционально)
- Дата (предзаполнена выбранным днём)
- Сохранить / Отмена

### 5. Настройки (иконка в углу)
- Экспорт данных → JSON файл
- Импорт данных ← JSON файл
- Сброс всех данных (с подтверждением)

## Дизайн

### Общее
- Тёмная тема, только тёмная
- Минимализм: никаких лишних элементов
- Закруглённые края везде (border-radius: 12-16px)
- Шрифт: Inter или системный sans-serif

### Цвета
```
background:     #1a1410  (глубокий тёмно-коричневый)
surface:        #242018  (тёмный янтарно-коричневый)
surface-raised: #2e2a1f  (приподнятая поверхность)
income:         #d4a843  (янтарно-золотой)
expense:        #c4603a  (терракотовый)
text-primary:   #f0e6d3  (тёплый кремовый)
text-secondary: #8a7a65  (приглушённый коричневый)
accent:         #d4a843  (янтарь как основной акцент)
border:         #3a3228  (едва заметная граница)
today:          #3d3420  (подсветка сегодняшнего дня)
```

### Типографика
- Текущий баланс: 36-40px, bold
- Числа в календаре: 11-12px
- Суммы событий: 16-18px

### Мобильный (приоритет)
- Минимальный touch target: 44px
- Свайп для переключения месяцев (touchstart/touchend)
- Форма добавления — bottom sheet снизу экрана
- Безопасные зоны для iPhone (env(safe-area-inset-*))

### Десктоп
- Максимальная ширина контента: 480px, центрирован
- Приложение выглядит как мобильное в центре экрана

## PWA
Обязательно:
