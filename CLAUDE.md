# Finance Manager — CLAUDE.md v2

## Цель
Личный финансовый менеджер. PWA. React. Мобильный + десктоп.
Понимать сколько денег есть сейчас, сколько можно тратить каждый день,
контролировать кредиты и накопления.

---

## Стек
- React (Vite)
- localStorage для хранения данных
- CSS Modules или Tailwind (на усмотрение, главное — чисто)
- Никакого бэкенда, никаких внешних API

---

## Структура данных

```js
// Текущий баланс (всегда редактируемый вручную)
currentBalance: number

// Разовые события календаря
events: [
  {
    id: string,
    date: "YYYY-MM-DD",
    type: "income" | "expense",
    amount: number,
    label: string,
    confirmed: boolean  // подтверждён ли факт платежа
  }
]

// Повторяющиеся платежи
recurringPayments: [
  {
    id: string,
    type: "income" | "expense",
    label: string,
    amount: number,           // базовая сумма (может плавать)
    frequency: "monthly" | "weekly" | "biweekly",
    anchorDate: "YYYY-MM-DD", // опорная дата для расчёта следующего платежа
    flexAmount: boolean,      // true = сумма может отличаться при подтверждении
    flexDate: number,         // допустимый разброс дней (например ±3)
    category: "income" | "expense" | "credit" | "savings"
  }
]

// Кредиты
credits: [
  {
    id: string,
    label: string,            // название (например "Тинькофф Платинум")
    totalDebt: number,        // текущий общий долг
    limit: number,            // кредитный лимит
    subPayments: [
      {
        id: string,
        label: string,        // например "Обычные траты", "Переводы"
        amount: number,       // сумма долга по этому виду
        interestRate: number  // процент годовых
      }
    ],
    minPayment: number,       // минимальный ежемесячный платёж
    paymentDate: number       // день месяца для оплаты
  }
]

// Накопительные счета и вклады
savings: [
  {
    id: string,
    label: string,            // название (например "Подушка безопасности")
    balance: number,          // текущий баланс
    interestRate: number,     // процент годовых (0 если нет)
    goal: number | null       // целевая сумма (опционально)
  }
]

// Правила распределения дохода
distributionRules: [
  {
    id: string,
    label: string,            // например "На жизнь"
    percentage: number,       // например 50
    targetType: "expense" | "credit" | "savings",
    targetId: string | null   // к чему привязано (null = свободные деньги)
  }
]

// Подтверждения повторяющихся платежей
confirmations: [
  {
    recurringId: string,
    date: "YYYY-MM-DD",       // когда фактически подтверждён
    actualAmount: number,     // фактическая сумма (может отличаться от базовой)
    confirmed: boolean
  }
]
```

## Логика расчётов

### Баланс на конкретный день
```
баланс(day) = currentBalance
  + сумма подтверждённых income до и включая day
  + сумма запланированных income до и включая day (неподтверждённые)
  - сумма подтверждённых expense до и включая day
  - сумма запланированных expense до и включая day (неподтверждённые)
```

### Сколько можно тратить сегодня
```
свободные = баланс(сегодня)
           + будущие income до конца месяца
           - будущие обязательные expense до конца месяца (кредиты, постоянные)

доступно_в_день = свободные / оставшиеся дни в месяце включая сегодня
```

### Расчёт следующей даты повторяющегося платежа
```
monthly:   следующий anchorDate.day в текущем или следующем месяце
weekly:    anchorDate + N * 7 дней
biweekly:  anchorDate + N * 14 дней
```

## Экраны и навигация

Нижняя навигация (tab bar) с 4 вкладками:

### 1. Главный экран (Today)
- Текущий баланс — большая цифра, кликабельная для редактирования
- Сколько можно потратить сегодня
- Ближайшие 7 дней: список запланированных доходов и расходов
- Требующие подтверждения платежи (если есть) — выделены отдельно
- Быстрая кнопка добавить разовое событие

### 2. Календарь
- Месячный вид, все дни месяца
- Переключение месяцев: стрелки + свайп на мобильном
- На каждом дне:
  - Точки или иконки если есть события
  - Прогнозируемый остаток на этот день
- Сегодня — визуально выделен
- Дни с отрицательным балансом — выделены терракотовым
- Клик на день → детальный вид дня

### 3. Детальный вид дня (bottom sheet или отдельный экран)
- Список всех событий дня (разовые + повторяющиеся)
- Для каждого события: лейбл, сумма, статус подтверждения
- Кнопка подтвердить платёж (с возможностью указать точную сумму)
- Кнопка добавить событие
- Итог: остаток на конец дня

### 4. Обязательства (Obligations)
Три таба внутри экрана:

**Повторяющиеся платежи**
- Список всех recurring payments
- Для каждого: название, сумма, периодичность, следующая дата
- Добавить / редактировать / удалить

**Кредиты**
- Карточка каждого кредита: название, общий долг, лимит, прогресс-бар
- Внутри карточки — субплатежи с процентами
- Минимальный платёж и дата списания
- Добавить / редактировать

**Накопления**
- Карточка каждого счёта: название, баланс, цель, прогресс-бар
- Процент годовых если есть
- Добавить / редактировать

### 5. Настройки
- Распределение дохода (правила в процентах, сумма правил = 100%)
- Экспорт данных → JSON
- Импорт данных ← JSON
- Сброс всех данных (с подтверждением)

## Подтверждение платежей

Когда наступает дата повторяющегося платежа:
- На главном экране появляется карточка "Требует подтверждения"
- Пользователь нажимает подтвердить
- Если flexAmount: true — открывается инпут с предзаполненной базовой суммой
- После подтверждения: запись в confirmations, баланс пересчитывается

Неподтверждённые платежи отображаются как "запланированные" (пунктир или приглушённый цвет).

---

## Распределение дохода

При подтверждении income:
- Система показывает как распределится сумма согласно правилам
- Например: 50% → на жизнь, 40% → на кредиты, 10% → в накопления
- Пользователь может принять или скорректировать
- После принятия: автоматически создаются платежи или пополнения счетов

## Дизайн

### Общее
- Тёмная тема, только тёмная
- Минимализм: никаких лишних элементов
- Закруглённые края везде (border-radius: 12-16px)
- Шрифт: Inter или системный sans-serif

### Цвета
```
background:     #1a1410  (глубокий тёмно-коричневый)
surface:        #242018  (тёмный янтарно-коричневый)
surface-raised: #2e2a1f  (приподнятая поверхность)
income:         #d4a843  (янтарно-золотой)
expense:        #c4603a  (терракотовый)
credit:         #a05c3a  (приглушённый терракот для кредитов)
savings:        #6a9e6f  (приглушённый зелёный для накоплений)
text-primary:   #f0e6d3  (тёплый кремовый)
text-secondary: #8a7a65  (приглушённый коричневый)
accent:         #d4a843  (янтарь как основной акцент)
border:         #3a3228  (едва заметная граница)
today:          #3d3420  (подсветка сегодняшнего дня)
pending:        #4a4030  (фон неподтверждённых платежей)
```

### Типографика
- Текущий баланс: 36-40px, bold
- Числа в календаре: 11-12px
- Суммы событий: 16-18px

### Мобильный (приоритет)
- Минимальный touch target: 44px
- Свайп для переключения месяцев (touchstart/touchend)
- Формы и детали — bottom sheet снизу экрана
- Безопасные зоны для iPhone (env(safe-area-inset-*))
- Нижняя навигация выше safe area

### Десктоп
- Максимальная ширина контента: 480px, центрирован
- Приложение выглядит как мобильное в центре экрана

## Приоритет разработки
1. Структура данных + localStorage (все сущности)
2. Главный экран (Today) — баланс + ближайшие события
3. Календарь с прогнозом остатков
4. Повторяющиеся платежи — добавление и расчёт дат
5. Подтверждение платежей с корректировкой суммы
6. Кредиты — структура и субплатежи
7. Накопления
8. Распределение дохода
9. Импорт / экспорт
10. PWA манифест + service worker
11. Свайп на мобильном
